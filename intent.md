# 意图工程洞察

从工作语境（context）中提炼的关键洞察。

## 1. 意图工程是认知工程在高阶认知层面的专门分支

意图工程补足了认知工程长期忽视的一个缺口：认知工程大量研究"如何支持已知目标的决策与执行"，但很少系统研究"目标/意图本身是如何从模糊变得清晰、可验证的"。

意图工程的核心认识论前提——**意图的不可全交割性**——为其提供了独立的哲学基础：人类意图永远无法被完整地交割给任何外部系统（包括AI），结构化契约只是意图的近似映射。

来源：`data/context/intent.md`、`data/context/index.md`

## 2. 情境-心智模型-意图 三元组是可计算的认知结构

情境（S）→ 心智模型（M）→ 意图（I）三者构成可计算的递进关系：

- 情境触发心智模型的调用与更新
- 心智模型用于形成与修正意图
- 意图指导行动以改变情境

在大模型语境下，这一结构可被实例化：情境对应上下文，意图对应指令/目标函数，心智模型对应可调用的外部知识模块或思维链规则。

来源：`data/context/index.md`

## 3. S-M-I 三元组的大模型计算框架

大模型计算 S-M-I 三元组的过程分为四步：

1. **接收**：大模型接收情境（S）和意图（I）
2. **调用**：根据 S 和 I，检索或提示大模型激活特定的心智模型（M）
3. **推理**：大模型基于"在情境 S 下，使用心智模型 M，如何行动才能达成意图 I"进行推理生成
4. **输出**：输出可行动的结果（预测、建议、指令）

边界条件：
- 大模型不是硬编码执行 f(s,i,m)=a 函数，而是通过上下文学习进行软性计算
- 心智模型需外显为文本描述，大模型没有内隐的固定心智模型
- 目前更适合模拟与设计，而非直接控制实时系统

来源：`data/context/index.md`

## 4. 意图契约的结构化骨架

一份可执行的意图契约至少包含五个构件：

| 构件 | 说明 |
|------|------|
| 本体条款 | 核心逻辑、不变约束、因果关系 |
| 表达条款 | 风格/调性、媒介约束、实现偏好 |
| 校验规则 | 自洽性条件、与原始意图的对齐标准 |
| 版本元数据 | 演化历史、合并记录 |

契约采用结构化格式（JSON、YAML 或 DSL），可被机器解析和校验。

来源：`data/context/intent.md`

## 5. D-V-C-E 闭环是通用的认知方法论

Discovery → Validate → Content → Evolution 四阶段循环不仅适用于意图工程，也可作为通用的认知工程方法论框架。其核心洞见是：**演化以合并与重组为主要操作**，避免琐碎的增量修补，防止碎片化。

来源：`data/context/intent.md`

## 6. 意图工程 vs. 提示词工程

| 维度 | 提示词工程 | 意图工程 |
|------|-----------|---------|
| 核心关注 | 如何造句让 AI 理解 | 如何让自身意图变得清晰可校验 |
| 意图状态 | 隐性，靠试错逼近 | 显性，提炼为契约 |
| 容错率 | 低，提示词差异影响巨大 | 高，契约提供强约束 |
| 人机关系 | 人类像祈求神谕的祭司 | 人类像审阅蓝图的总工程师 |
| 适用边界 | 单次、简单的生成任务 | 复杂、多步、跨领域系统性任务 |

来源：`data/context/intent.md`
