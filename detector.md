# 认知负担检测器

## 核心目标

一眼看出文本哪里"难读"。

---

## 工作流程

两阶段流水线：**门禁 → 诊断**。门禁没过的不进诊断。

```
输入文本
  ↓
[门禁] 信息密度 + 逻辑跳跃度
  ├── 放行 → 跳过诊断，直接用
  ├── 打回 → 告知原因，不改了
  └── 标记拆解 → 进诊断
                        ↓
            [诊断] 6 项指标打分
                        ↓
                  输出修改建议
```

---

## 阶段一：门禁（要不要读？）

先判断一段文本值不值得花脑力。只看两样：

### 信息熵

信息密度有多高。熵低 = 废话多，熵高 = 术语密集。

- **熵过低** → 打回（全是废话）
- **熵过高** → 标记拆解（过于浓缩，需要拆开）
- 计算方法：字符级熵（统计字符概率分布）或用 gzip 压缩比估算

### 逻辑跳跃度

句与句之间推理链条断没断。跳跃高 = 需要读者自己脑补"因为……所以……"。

- **跳跃高且没连接词** → 打回或标记拆解
- 计算方法：连接词密度检测（"因为""所以""然而"等），或用向量余弦算相邻句相似度

### 门禁输出矩阵

| 信息密度 | 逻辑跳跃 | 判定 | 动作 |
|----------|----------|------|------|
| 低 | 低 | 流水账 | 打回 |
| 低 | 高 | 不知所云 | 打回 |
| 高 | 低 | 硬核干货 | 放行 |
| 高 | 高 | 密度大跳跃快 | 标记拆解 |

---

## 阶段二：诊断（怎么改？）

被门禁标记"拆解"的文本才进这步。跑 6 项检查，每项给出具体位置和修改建议。

### 6 项指标

| 指标 | 看什么 | 权重 |
|------|--------|------|
| 逻辑跳跃 | 上下段不连贯？ | 0.25 |
| 无效表格 | 数据很少却强行列表？ | 0.15 |
| 废话连篇 | 车轱辘话反复说？ | 0.15 |
| 套娃标题 | 标题层级超过 3 层？ | 0.10 |
| 硬转折 | 段首没有过渡词？ | 0.20 |
| 概念轰炸 | 一段里新词太多？ | 0.15 |

### 算分方法

每项 0-1 分。总分 = 各项分数加权求和。

- **< 0.30**：好文，直接用。
- **0.30 - 0.55**：偏高，挑高分项改。
- **> 0.55**：重写，别修了。

### 怎么实现

主要靠规则：数标题层级、算文本相似度、查过渡词表。快且免费。

实在不懂再问 AI：比如判断"这段话逻辑到底断没断"，才调用 LLM。

### 怎么改

根据报错直接改：

> **报错：第 12 段逻辑跳跃** → 行动：加一句过渡句。

> **报错：表格无效** → 行动：把表格变成一句话。

> **报错：概念轰炸** → 行动：拆成两段，或加个例子。

## 实测结果（quanttide-data 文档集）

| 文档 | 评分 | 问题 |
|------|------|------|
| README.md | 100% (低) | 无 |
| why.md | 66% (中) | 概念密度 0%, 过渡词 0% |
| blueprint.md | 67% (中) | 概念密度 0%, 过渡词 6% |
| design-philosophy.md | 73% (中) | 过渡词 0%, 概念密度 40% |
| engineering-standards.md | 68% (中) | 过渡词 0%, 概念密度 10% |

**发现：** 所有文档的共同痛点是**概念密度过高**（每段引入 5-14 个新术语）和**缺乏过渡词**。标题层级和表格使用正常。

## 扩展

### 代码认知负担检测

代码的认知负担主要来自三方面：

- **嵌套深度**：if → for → if → try 嵌套超过 4 层，大脑需要维护多个栈帧
- **圈复杂度**：函数逻辑分支太多
- **命名熵**：`data`、`temp`、`result`（低熵但高混淆）与 `userAuthenticationManager`（高熵）混在一起

检测到深层嵌套时，建议拆函数或用卫语句提前返回。

### AI 输出常见毛病清单

| 毛病 | 表现 |
|------|------|
| 正确的废话 | "在当今数字化浪潮中"——信息量为零 |
| 假过渡 | "此外"后面跟的内容没有逻辑关系 |
| 缺例子 | 全程抽象概念，不给具体场景 |
| 堆砌术语 | 不用大白话，非要"赋能""闭环""颗粒度" |
| 开头公式化 | "随着……的发展""近年来……"等空泛开头 |
| 假精确 | "大约""基本上"高频堆叠 |

### 阈值校准

不要猜阈值。找 3 篇最轻松的文档和 3 篇最头疼的文档，算平均分设为低/高阈值。
