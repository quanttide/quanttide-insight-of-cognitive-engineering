# Journal data pyramid

## 洞察

数据按**变化频率**组织，从慢到快：

```
schemas      ← 慢变化（周级）
situations   ← 中变化（周级）
intentions   ← 快变化（天级）
thoughts     ← 最快（每天追加）
```

越往上越凝固，越往下越活跃。打开文件从最慢的开始读——如果 schema 跟上个月一样，跳过就行。

## 来由

gallery 仓库降级为 journal 时，探讨字段排序。最初的直觉是按"稳定度"排，但错了——四个类型都在变，只是速度不同。区别不是"变 vs 不变"，是变化频率的梯度。

## 影响

- 文件结构：`schemas` 排第一个，`thoughts` 最后一个
- 读写模式：读从上往下（看框架 → 看状态 → 看意图），写从下往上（原始日志 → 提炼意图 → 更新情境 → 沉淀模式）
- 不要把 `thoughts` 当作"临时草稿"，它只是变化最快的数据层
