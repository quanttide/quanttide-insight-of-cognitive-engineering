# 检测器扩展：代码与注释

## 问题

当前 6 项诊断指标只适用于中文 Markdown 散文。

用在代码上时暴露三个盲区：

1. **没有代码指标** — 函数长度、嵌套深度、圈复杂度、命名熵，全没测
2. **没有注释指标** — 80 行无注释的函数和 80 行每段有注释的函数，评分一样
3. **单函数太长** — 检测器自己就有一个 80 行的 `logic_jump`，但测不出来

---

## 扩展方案

三个新指标，全部归入**诊断阶段**。门禁不变（信息熵 + 逻辑跳跃度对代码和文本都有效）。

### 指标一：函数长度

| 项目 | 说明 |
|------|------|
| 阶段 | 诊断 |
| 看什么 | 代码中每个函数的行数 |
| 阈值 | > 40 行扣分，> 80 行大幅扣分 |
| 实现 | 按 `fn` 关键字分割代码段，数起始到下一个 `fn` 或文件结束的行数 |

### 指标二：注释密度

| 项目 | 说明 |
|------|------|
| 阶段 | 诊断 |
| 看什么 | 注释行占总行数的比例，以及公开 API 是否有文档注释 |
| 阈值 | 注释行 < 5% 扣分；公开函数缺失 `///` 文档注释扣分 |
| 实现 | 统计 `//` 和 `///` 开头的行数占比；用正则匹配 `pub fn` 前的 `///` 块 |

### 指标三：结构复杂度

| 项目 | 说明 |
|------|------|
| 阶段 | 诊断 |
| 看什么 | 嵌套深度 + 圈复杂度 + 命名熵，三者合并 |
| 阈值 | 嵌套 > 4 层扣分；单函数圈复杂度 > 10 扣分；命名熵异常（全低或全高）扣分 |
| 实现 | 缩进层级数检测；匹配 `if/for/while/match` 计数；标识符长度分布统计 |

---

## 工作流变化

```
输入       门禁                    诊断
文本  ──→  熵 + 跳跃度  ──→  6 项散文指标
代码  ──→  熵 + 跳跃度  ──→  3 项代码指标
混合  ──→  熵 + 跳跃度  ──→  两项都跑
```

门禁对文本和代码统一用熵 + 跳跃度。门禁通过后，按输入类型路由到不同的诊断指标集。

## 类型检测

在门禁后加一步简单判断：

- 文件以 `.rs`, `.py`, `.js`, `.ts`, `.go` 结尾 → 代码
- 文件以 `.md`, `.txt`, `.rst` 结尾 → 文本
- 其他 → 按内容第一行是否有 `fn`, `def`, `function`, `import` 等关键词判断
